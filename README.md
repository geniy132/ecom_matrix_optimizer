# Demand and Margin Analyzer (E-commerce Data Tools)

Автономный аналитический инструмент на Python для автоматизации прогнозирования товарного спроса и маржинального анализа ассортиментной матрицы на маркетплейсах.

## 📌 Описание проекта
Скрипт разработан для оптимизации ежедневных задач категорийных менеджеров и экономистов в e-commerce. Программа полностью автоматизирует обработку выгрузок динамики продаж из Excel, минимизируя человеческий фактор и ручной ввод формул.

### Выполняемые задачи:
1. **Data Cleaning (Очистка данных):** Динамический поиск шапки таблицы, автоматическое определение столбцов с временными рядами дат, строгая фильтрация строк итогов и системного мусора по регулярным выражениям (`re`).
2. **Прогнозирование спроса:** Расчет среднесуточной скорости продаж и автоматическое вычисление штучной потребности в закупках на будущие периоды (Март, Апрель, Май отдельно).
3. **Маржинальный аудит ассортимента:** Анализ общей маржинальной прибыли за период и генерация развернутых бизнес-рекомендаций по оптимизации матрицы (выделение драйверов прибыли, потенциальных "звезд" и балластовых SKU).

## 🛠 Технологический стек
* **Язык:** Python 3.10+
* **Библиотеки:** Pandas, OpenPyxl, Regex (`re`)
* **Сборка:** PyInstaller (для компиляции в независимый `.exe` формат)

## 🚀 Инструкция по запуску

### Вариант 1: Запуск готовой утилиты (без установленного Python)
1. Перейдите в папку `dist/` (или скачайте скомпилированный файл).
2. Поместите исходный Excel-файл с данными (переименованный в `data.xlsx`) в одну директорию с файлом `analyzer.exe`.
3. Запустите `analyzer.exe`. Результат запишется в файл `Итоговый_анализ_ТЗ.xlsx`.

### Вариант 2: Запуск из исходного кода
1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com
   ```
2. Установите зависимости:
   ```bash
   pip install pandas openpyxl
   ```
3. Положите файл `data.xlsx` в корень проекта и запустите:
   ```bash
   python analyzer.py
   ```

## 📊 Логика генерации бизнес-рекомендаций
Критерии автоматических выводов привязаны к динамической медиане скорости продаж всей матрицы:
* **Драйвер прибыли:** Высокий спрос + Маржа ≥ 15%. (Инвестиции в SEO, контроль остатков).
* **Потенциальная звезда:** Низкий спрос + Маржа ≥ 15%. (Тест снижения цены, проверка инфографики).
* **Балласт:** Маржа < 12% + Низкий спрос. (Распродажа остатков и вывод из матрицы).


## 📂 Архитектурная структура проекта
Проект реализован в рамках концепции модульного программирования для повышения читаемости и упрощения модульного тестирования (Unit Testing):
* `utils.py` — вспомогательные инструменты для парсинга, динамического поиска шапок и регулярных выражений очистки данных (ETL/Data Cleaning).
* `core.py` — ядро системы вычислений, содержащее алгоритмы математического прогнозирования и матричного анализа.
* `main.py` — оркестратор (точка входа), координирующий последовательный запуск пайплайна обработки данных.
