# ECOM Matrix Optimizer

Модульный аналитический инструмент на Python для автоматизации прогнозирования товарного спроса и маржинального анализа ассортиментной матрицы брендов на маркетплейсах.

## 📌 Описание проекта
Скрипт разработан для оптимизации аналитических задач коммерческих департаментов в сфере E-commerce. Программа полностью автоматизирует обработку сырых выгрузок посуточной динамики продаж из Excel, исключая человеческий фактор и необходимость ручного ввода формул.

### Выполняемый функционал:
1. **Умный ETL-пайплайн:** Многокритериальный поиск шапки (игнорирует вводные тексты) и интеллектуальный приоритетный поиск столбца номенклатуры (сначала ищет «Полное название», затем синонимы).
2. **Всеядный парсинг дат:** Автоматически распознает временные ряды продаж в любых форматах (строки, объекты `Timestamp` или числовые коды дат Excel). 
3. **Адаптивный календарный прогноз:** Скрипт находит крайнюю дату выгрузки и рассчитывает штучную закупку на 3 последующих месяца вперед, автоматически вычисляя дни по календарю (с учетом високосных лет).
4. **Маржинальный аудит и бизнес-дизайн:** Расчет прибыли за период и автоматическая генерация инсайтов на основе скользящей медианы матрицы. Итоговый файл оформляется в корпоративном стиле (автоширина колонок, пастельное условное форматирование строк).
5. **Configuration Separation:** Все пороговые значения маржи и текстовые шаблоны отчетов полностью вынесены во внешний файл `config.json`.

## 📂 Архитектура и структура проекта

Проект спроектирован по принципам SOLID и разделен на изолированные слои: интерфейс (меню), оркестратор (конвейер), discovery (локализация данных), etl (очистка) и операции (бизнес-логика).

```text
ecom_matrix_optimizer/
│
├── main.py                    # Главный файл-оркестратор (входная точка, консольное меню, сборка Excel-вывода)
├── config.json                # Декларативный бизнес-конфиг (настройки меню, маржинальные пороги, цвета CSS)
├── config_loader.py           # Модуль загрузки глобальных параметров и настроек из JSON
│
├── core.py                    # SOLID-реестр и диспетчер распределения команд по номерам сценариев
│
├── discovery.py               # Интеллектуальный Layout-сканер (автопоиск Листа 1 или Листа 3, синонимы шапок)
├── styler.py                  # Классический openpyxl-движок (вертикальное центрирование строк, разделители тысяч, пастельные цвета)
├── utils.py                   # Служебные ETL-инструменты (очистка текстового мусора, извлечение векторов дат)
│
├── operations_forecast.py     # [Операция 1] Прогнозирование спроса, расчет коэффициентов и аудит матрицы
├── operations_transactions.py # [Операция 2] Агрегация штучных объемов и выручки по логам транзакций под список SKU
└── operations_promo.py        # [Операция 3] Анализ промо-акций (американские даты ММ/ДД/ГГГГ, расчет Sales Lift и каннибализации)
```

### ⚙️ Описание ключевых аналитических сценариев

1. **`operations_forecast.py` [Операция 1]**: Проводит маржинальный ABC-аудит базовой матрицы. Рассчитывает весенние прогнозы спроса и размечает категории («Драйверы», «Звезды», «Балласт»).
2. **`operations_transactions.py` [Операция 2]**: Векторно сжимает сырой журнал транзакций маркетплейса. Группирует штуки и рубли строго по списку артикулов, переданных из буфера обмена.
3. **`operations_promo.py` [Операция 3]**: Реализует адаптивный гибридный парсер американского формата дат. Вычисляет точный индекс `Sales Lift` (с сохранением знака минус) и диагностирует маркетинговые эффекты замещения трафика внутри бренда.

## 🛠 Технологический стек
* **Язык:** Python 3.10+ (строгое соответствие стандарту PEP 8)
* **Библиотеки:** Pandas, OpenPyxl, Regex (`re`)
* **Сборка:** PyInstaller (компиляция в независимый исполняемый `.exe` формат)

## 🚀 Инструкция по развертыванию и запуску

### Запуск из исходного кода:
1. Клонируйте репозиторий, установите зависимости и запустите оркестратор:
   ```bash
   git clone https://github.com
   pip install -r requirements.txt
   python main.py
   ```
2. Введите полный путь к вашему файлу (в Windows: нажать на файл через **ПКМ -> Копировать как путь** и вставить в консоль). Скрипт автоматически очистит кавычки, проведет расчеты и сохранит оформленный файл `Итоговый_анализ.xlsx` в папку к исходному документу.
